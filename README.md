# Project_1 — Natours Landing Page


**Description**
- This repository contains the Natours landing page UI built using SASS (organized under `sass/`) and compiled to CSS (under `css/`).

**Prerequisites**
- Node.js and npm installed (tested with Node 14+ / npm 6+).

**Install**
Run in project root:

```bash
npm install
```

**Available npm scripts**
- `npm run watch:sass` — compile `sass/main.scss` to CSS and watch for changes (writes to `css/style.comp.css` by default in this repo).
- `npm run compile:sass` — compile once.
- `npm run devserver` — runs `live-server` to serve the project folder.
- `npm start` — runs devserver + watch in parallel (if `npm-run-all` is available).
- `npm run build:css` — runs the CSS build pipeline (compile, concat, prefix, compress) if configured in `package.json`.

Note: The project originally used intermediate files (`style.concat.css`, `style.prefix.css`, etc.). If you prefer a single compiled file for development, change the `compile:sass` script to write directly to `css/style.css`.

**CSS files (what they are)**
- `css/icon-font.css` — icon font / @font-face declarations used by the site.
- `css/style.comp.css` — compiled output from Sass (`node-sass sass/main.scss css/style.comp.css`).
- `css/style.concat.css` — concatenated CSS (generated when using the `concat` step).
- `css/style.prefix.css` — autoprefixed CSS (generated by PostCSS + Autoprefixer).
- `css/style.css` — final/minified CSS served to the site (your HTML links to `css/style.css`).
- `css/test.css` — experimental or test styles (optional).

If you want a simpler workflow that produces only `css/style.css`, update `package.json` so `compile:sass` writes to `css/style.css` and remove/uninstall the `concat` step.

**Sass structure**
- `sass/abstracts/` — variables, mixins, functions (`_mixins.scss`, `_variables.scss`, etc.).
- `sass/base/` — base styles, typography, utilities.
- `sass/components/` — component styles (cards, buttons, forms, etc.).
- `sass/layout/` — layout-specific styles (header, footer, grid).
- `sass/pages/` — page-specific styles.

**Recent fixes and notes**
- Fixed a media-query syntax issue in `sass/abstracts/_mixins.scss` (the `respond()` mixin previously placed `@content` inline; now it uses proper media-query blocks). This prevents Sass compilation errors and makes responsive rules behave correctly.
- I compiled the Sass once during cleanup; `css/style.comp.css` was written.

**Recommended workflow**
1. Install dependencies: `npm install`
2. Compile once: `npm run compile:sass`
3. During development: `npm run watch:sass` and `npm run devserver` (or `npm start`)

**If you want to produce the final `css/style.css` directly**
Edit `package.json` and set the compile output path:

```json
"compile:sass": "node-sass sass/main.scss css/style.css"
```

Then run:

```bash
npm run compile:sass
```
